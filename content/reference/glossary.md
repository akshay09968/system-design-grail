# Glossary

One-sentence definitions for every load-bearing term, each linked to the page that teaches it properly. Use search (++s++) — this page exists to be landed on.

## A–C

**ACID** — atomicity, consistency, isolation, durability: the single-node transaction contract. → [Transactions](../data/transactions.md)
**Active-active / active-passive** — redundancy where all nodes serve vs. a standby awaiting failover. → [Reliability](../foundations/reliability-availability.md), [Multi-region](../devops/multi-region.md)
**Amdahl's law** — speedup is capped at 1/serial-fraction, no matter the node count. → [Scalability](../foundations/scalability.md)
**Anti-entropy** — background reconciliation (often Merkle-tree diff) that converges replicas. → [Replication](../data/replication.md)
**Anycast** — one IP announced from many sites; routing delivers users to the nearest. → [DNS](../networking/dns.md)
**At-least-once / at-most-once** — delivery contracts: retries-with-duplicates vs. fire-and-forget. → [Delivery semantics](../messaging/delivery-semantics.md)
**Backpressure** — a slow consumer's slowness propagating upstream visibly instead of pooling invisibly. → [Async fundamentals](../messaging/async-fundamentals.md)
**Blast radius** — the scope of damage when a component fails; a designable quantity. → [Reliability](../foundations/reliability-availability.md)
**Bloom filter** — compact probabilistic "definitely not present" test; false positives, never false negatives. → [Storage engines](../data/storage-engines.md)
**Bulkhead** — resource partitioning (pools, quotas) so one dependency's failure can't drown the rest. → [Resilience](../distributed/resilience.md)
**Burn rate** — error-budget consumption speed; the basis of modern alerting. → [SLOs](../observability/slos.md)
**CAP theorem** — during a partition, choose consistency or availability; not a triangle. → [CAP & PACELC](../foundations/cap-pacelc.md)
**Cardinality** — distinct label/value combinations; the master constraint of telemetry cost. → [Observability](../observability/fundamentals.md)
**CDC (change data capture)** — tailing a database's log to stream every change elsewhere. → [Analytics](../data/analytics.md)
**Cell architecture** — partitioning the whole stack into independent failure-contained units. → [Reliability](../foundations/reliability-availability.md)
**Circuit breaker** — stop calling a failing dependency; fail fast, probe for recovery. → [Resilience](../distributed/resilience.md)
**Consistent hashing** — ring-based key placement where membership change moves only K/N keys. → [Partitioning](../data/partitioning.md)
**CQRS** — separate write model from read projections, each shaped for its job. → [Event-driven](../messaging/event-driven.md)
**CRDT** — data structure whose replicas merge automatically and commutatively. → [Replication](../data/replication.md)

## D–H

**Dead-man's switch** — an always-firing signal whose *silence* is the alert. → [Alerting](../observability/alerting.md)
**Deadline propagation** — remaining time-budget travels with the request; hops give up early. → [Resilience](../distributed/resilience.md)
**DLQ (dead-letter queue)** — quarantine for messages that exhaust retries; monitored, replayable. → [Async fundamentals](../messaging/async-fundamentals.md)
**Envelope encryption** — data encrypted with a DEK; DEK wrapped by a KMS-held KEK. → [Secrets](../devops/secrets-identity.md)
**Erasure coding** — k-of-n fragment redundancy: survive losses at ~1.5× overhead vs. 3× replication. → [Object storage](../data/object-storage.md)
**Error budget** — the allowed failure implied by an SLO; spendable on velocity. → [SLOs](../observability/slos.md)
**Event sourcing** — the append-only event log as truth; state as a rebuildable fold. → [Event-driven](../messaging/event-driven.md)
**Expand–contract** — schema/credential migration via add-alongside, dual-write, flip, remove-later. → [SQL at scale](../data/sql-at-scale.md)
**Fail static** — on control-plane loss, keep operating on last-known configuration. → [K8s architecture](../devops/kubernetes-architecture.md)
**Fan-out** — one action becoming N operations (writes, calls); the amplification to count. → [Thinking in systems](../foundations/thinking-in-systems.md)
**Fencing token** — monotonic token letting a resource reject stale lock-holders. → [Coordination](../distributed/coordination.md)
**Gossip** — peer-to-peer state dissemination; membership without a master. → [KV store](../case-studies/kv-store.md)
**Gray failure** — passing health checks while failing real work; the differential-observability failure. → [Failure modes](../distributed/failure-modes.md)
**Head-of-line blocking** — one stalled item blocking everything behind it (TCP, queues, partitions). → [HTTP](../networking/http.md)
**Hedged request** — duplicate a laggard request; take the first answer; tame the tail. → [Latency](../foundations/latency-throughput.md)
**HLC (hybrid logical clock)** — timestamps that look like wall time but respect causality. → [Time & ordering](../distributed/time-ordering.md)
**Hot key/partition** — one key's traffic exceeding its shard's capacity; the fleet average lies. → [Cache failure modes](../caching/failure-modes.md)

## I–P

**Idempotency** — doing it twice equals doing it once; the license for retries. → [Delivery semantics](../messaging/delivery-semantics.md)
**ISR (in-sync replicas)** — Kafka's caught-up replica set; the durability quorum. → [Kafka](../messaging/kafka.md)
**Jitter** — randomized delay that desynchronizes herds (retries, TTLs, reconnects). → [Resilience](../distributed/resilience.md)
**Lease** — a lock with an expiry; the only sane distributed lock. → [Coordination](../distributed/coordination.md)
**Linearizability** — the single-copy illusion: reads see the latest completed write. → [Consistency models](../foundations/consistency-models.md)
**Little's law** — concurrency = throughput × latency; the equation you'll use forever. → [Latency](../foundations/latency-throughput.md)
**Load shedding** — rejecting excess early and cheaply so accepted work completes. → [Resilience](../distributed/resilience.md)
**LSM tree** — write-optimized storage: memtable → immutable sorted files → compaction. → [Storage engines](../data/storage-engines.md)
**Metastable failure** — a system stuck broken after the trigger clears, sustained by a feedback loop. → [Failure modes](../distributed/failure-modes.md)
**MTTR** — mean time to recover; the availability lever you actually control. → [Reliability](../foundations/reliability-availability.md)
**MVCC** — multi-version concurrency control: readers see snapshots, writers make versions. → [Transactions](../data/transactions.md)
**Outbox pattern** — event row committed in the business transaction; relay publishes after. → [Distributed transactions](../data/distributed-transactions.md)
**PACELC** — partitions force A-vs-C; *else*, latency-vs-consistency — the everyday tax. → [CAP & PACELC](../foundations/cap-pacelc.md)
**Percentile (p99)** — the value 99% of observations fall under; how latency is honestly spoken. → [Latency](../foundations/latency-throughput.md)
**Power of two choices** — pick two random backends, send to the less loaded; near-optimal. → [Load balancing](../networking/load-balancing.md)

## Q–Z

**Quorum (R+W>N)** — overlapping read/write sets guaranteeing reads see acknowledged writes. → [Consistency models](../foundations/consistency-models.md)
**Raft** — the understandable consensus protocol: terms, elections, replicated log. → [Consensus](../distributed/consensus.md)
**Rate limiting** — the front door's contract; token bucket and friends. → [Rate limiting](../distributed/rate-limiting.md)
**RED / USE** — service metrics (rate, errors, duration) / resource metrics (utilization, saturation, errors). → [Metrics](../observability/metrics.md)
**Regional homing** — each tenant's writes owned by a home region; conflicts impossible by assignment. → [Multi-region](../devops/multi-region.md)
**RPO / RTO** — how much data you may lose / how long recovery may take. → [Multi-region](../devops/multi-region.md)
**Saga** — distributed transaction as local steps + compensating actions. → [Distributed transactions](../data/distributed-transactions.md)
**Shuffle sharding** — per-customer random node subsets; blast radius shrinks combinatorially. → [Reliability](../foundations/reliability-availability.md)
**Sidecar** — per-instance proxy carrying cross-cutting concerns (mTLS, retries, telemetry). → [Service mesh](../devops/service-mesh.md)
**SLI / SLO / SLA** — measurement / target / contract-with-money. → [SLOs](../observability/slos.md)
**Split brain** — two nodes both believing they lead; divergent histories; why fencing exists. → [Replication](../data/replication.md)
**Stale-while-revalidate / stale-if-error** — serve stale instantly while refreshing / when origin is down. → [CDN](../networking/cdn.md)
**Thundering herd / stampede** — synchronized demand (expiry, reconnect, retry) crushing a target. → [Cache failure modes](../caching/failure-modes.md)
**Tombstone** — a delete marker that must replicate before it can be forgotten. → [KV store](../case-studies/kv-store.md)
**TrueTime** — bounded-uncertainty clock intervals; commit-wait buys external consistency. → [Time & ordering](../distributed/time-ordering.md)
**Vector clock** — per-node counters detecting causality vs. concurrency. → [Time & ordering](../distributed/time-ordering.md)
**WAL (write-ahead log)** — append-first durability; the log the database is secretly made of. → [Storage engines](../data/storage-engines.md)
**Watermark** — a stream engine's claim that events older than T have (probably) arrived. → [Event-driven](../messaging/event-driven.md)
**Workload identity** — attested, short-lived credentials replacing distributed secrets. → [Secrets & identity](../devops/secrets-identity.md)
**Zero trust** — authenticate every request, least privilege, assume breach. → [Defense in depth](../security/defense-in-depth.md)
