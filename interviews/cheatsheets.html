    Cheat Sheets¶

    The morning-of review, everything on two screens: the numbers, the decision tables, and the pattern triggers. Each row links back to its full page — this is the index of the site's muscle memory, not a substitute for it.

    The numbers (full page)¶

    Latency Throughput anchors
    Memory ref ~100 ns Tuned API node 1–10k RPS
    NVMe read ~20–100 µs nginx static/proxy 50k+ RPS
    Same-AZ RTT ~0.5 ms Postgres (indexed) 10–50k QPS
    Cross-region RTT 30–150 ms Redis 100k+ ops/s/core
    Intercontinental ~200 ms Kafka partition ~10–20 MB/s
    TLS cold start 1–2 RTT One big box 128 cores / TBs RAM / 25 Gbps

    Recipes: QPS ≈ DAU × actions ÷ 10⁵, peak 3×. Storage = rate × size × 3 (replication) × retention. Little's law: concurrency = throughput × latency. Availability: serial multiplies, parallel multiplies the unavailabilities. Fan-out tail: P(all N fast) = pᴺ — 0.99¹⁰⁰ ≈ 37%.

    Cost anchors (full page): compute ~$0.04/vCPU-hr · egress ~$0.05–0.09/GB · NAT ~$0.045/GB · cross-AZ ~$0.01–0.02/GB · S3 $0.023 → $0.001/GB-mo by tier.

    Decision tables¶

    Datastore by access pattern (full page)

    Pattern Reach for
    Relational, transactional, ad-hoc queries PostgreSQL + the ladder
    Key lookup at scale, sessions, counters KV / Redis / DynamoDB
    Single-partition range reads (feeds, time series) Wide-column, query-first modeling
    Full-text search Search index (fed by CDC)
    Relationship traversals ≥3 deep Graph — but start with SQL joins
    Blobs Object storage + CDN, metadata in DB
    Analytics Columnar: lake/warehouse via CDC — never the OLTP primary

    Consistency by data class (full page): money/inventory → linearizable + idempotency keys · own-content visibility → read-your-writes (leader reads or LSN tokens) · social/catalog → eventual + bounded staleness · counters/presence → eventual, approximate.

    Async & delivery (full pages): user waits on it → sync; else queue. Tasks → competing consumers; events/fan-out/replay → Kafka-shaped log. Delivery = at-least-once + idempotent consumers, always say so. Ordering → per-key partitions, never global.

    Caching sentence (full page): cache-aside + TTL-with-jitter + delete-on-write + single-flight + negative caching + L1/L2 for hot keys — six clauses, recite as one.

    Resilience sentence (full page): budget timeouts with deadline propagation; edge-only retries, 3 max, exponential + full jitter, under a retry budget; per-dependency breakers with stale-serving fallbacks; bulkheaded pools; priority load-shedding at admission.

    Scaling ladder (SQL / general): optimize queries → pool → cache → replicas → bigger box → partition tables → split by function → shard (pre-split logical shards, Snowflake IDs, the key chosen for cardinality + load + query-presence + isolation).

    Multi-region (full page): ask why (DR/latency/residency) → the ladder (multi-AZ → backups → pilot light → warm → active-passive → active-active) → data via regional homing → static stability + shared-fate audit.

    Pattern triggers (hear X → reach for Y)¶

    Trigger in the prompt Pattern
    "Viral / celebrity / hot item" Hot key: L1 cache, salting, dedicated shard
    "Users see their own post instantly" Read-your-writes mechanisms
    "Exactly once" At-least-once + idempotency, say the truth
    "Only one worker may..." The ladder: ownership > CAS > lease+fencing > lock
    "Feed / timeline" Push/pull hybrid fan-out
    "Upload files/video" Presigned URLs, multipart, CDN
    "Real-time updates" SSE if one-way; WebSockets + pub/sub backbone if duplex
    "Search-as-you-type" Typeahead: prefix structures + cached top-K
    "Count views/likes at scale" Approximate: HLL, sharded counters, batch-flush
    "Schedule / delayed jobs" ZSET or delay queues + reclaim leases
    "Third-party integration" Webhooks signed + retried + reconciliation poll
    "Now make it 10×" The bottleneck sequence, narrated in order
    "What if X dies?" The four questions + degraded-mode menu
    "Deploy this safely" Flag → canary + analysis → waves → expand-contract

    The operability close (your signature)¶

    "Observability: RED histograms per service, queue age, end-to-end traces. SLO: burn-rate alerts, multi-window. Rollout: flag-gated, canaried by cell, auto-rollback. Failure: the two scariest boxes have named degraded modes, and the failover is game-day'd quarterly."

    Four sentences. Almost nobody else in the loop will say them. That's the edge — go collect it.
